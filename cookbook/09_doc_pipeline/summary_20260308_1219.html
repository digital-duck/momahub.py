Overview

This document presents SPL (Structured Prompt Language), a declarative query language for large language models (LLMs) that treats LLMs as generative knowledge bases and their context windows as constrained resources. SPL provides explicit token budgeting, automatic query optimization, and native integration of retrieval-augmented generation (RAG) and persistent memory.

Key Points

• SPL is a declarative query language for managing LLM context windows.

• SPL treats LLMs as generative knowledge bases, enabling the synthesis of novel responses.

• The language includes explicit token budgeting, automatic query optimization, and native integration of RAG and persistent memory.

• SPL provides transparency through an EXPLAIN mechanism, allowing developers to inspect and reason about token allocation before execution.

Details

SPL's central conceptual innovation is treating LLMs as generative knowledge bases, which enables the synthesis of novel responses. This perspective is not purely theoretical, but rather a reflection of how practitioners who build AI applications daily find that the mental model of "LLM as database" is the most productive frame for understanding what LLMs actually do.

SPL's dual-clause architecture (SELECT + GENERATE) directly mirrors this distinction: SELECT populates the contextual knowledge, and GENERATE triggers the full synthesis process. This separation enables the optimizer to treat context gathering and generation as distinct phases with different resource characteristics, leading to better budget allocation.
